# Simulation Lab - Curriculum Plan

## Path Overview

**Path ID**: simulation-lab
**Difficulty**: Beginner to Intermediate
**Duration**: 6-8 weeks
**Category**: Tools
**Target Audience**: Ages 11+, requires Block Coding

## Description

Simulation Lab teaches students to use virtual robotics environments for testing and debugging programs without physical hardware. Students learn to program and test in simulated environments, practice competition challenges, and develop debugging skills in a safe, cost-effective setting.

By completing this path, students can:
- Use virtual robot simulators effectively
- Debug programs in simulation
- Practice navigation and missions virtually
- Test on competition fields
- Develop reliable code before hardware deployment

## Prerequisites

- Block Coding (basic programming logic)

## Level Structure

### Level 1: Simulation Basics (Weeks 1-2)
Introduction to virtual robotics environments.

**Modules:**
- Introduction to Simulation (benefits, tools, virtual SPIKE hub)
- Basic Virtual Testing (setup, running programs, output interpretation)

**Assessment**: Run successful virtual program

**Project**: Virtual robot movement challenge

### Level 2: Simulated Movement and Environments (Weeks 3-4)
Practicing navigation and challenges virtually.

**Modules:**
- Movement Simulation (motor models, sensor simulation, physics)
- Challenge Maps (available maps, line following, object detection)

**Assessment**: Complete virtual navigation challenge

**Project**: Virtual mission completion

### Level 3: Advanced Simulation Scenarios (Weeks 5-6)
Complex testing and troubleshooting.

**Modules:**
- Debugging in Simulation (step execution, breakpoints, logging)
- Realistic Scenarios (multi-robot, environmental variation, sensor noise)

**Capstone**: Complex simulation mission with debugging

## Key Topics

### Simulation Environments

### Debugging Techniques
- Step-by-step execution
- Breakpoints and watches

### Practical Applications
- Testing before hardware deployment
- Rapid iteration without hardware costs
- Environmental variation testing
- Team practice without full kit
- Competition preparation

**Summative:**
