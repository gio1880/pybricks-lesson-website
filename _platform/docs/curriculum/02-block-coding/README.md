# Block Coding - Curriculum Plan

## Path Overview

**Path ID**: block-coding
**Difficulty**: Beginner
**Duration**: 6-8 weeks
**Category**: Programming
**Target Audience**: Ages 10+, requires Robotics Foundations

## Description

Block Coding introduces visual programming for robots using drag-and-drop block languages. Students learn to control LEGO robots using SPIKE or similar platforms, mastering sequences, loops, conditionals, and event-driven programming. This path bridges mechanical understanding with logical thinking.

By completing this path, students gain:
- Proficiency with block-based visual programming
- Understanding of program sequences and logic flow
- Ability to program loops and conditional logic
- Event-driven programming skills
- Ability to program complete robot behaviors

## Prerequisites

- Robotics Foundations (understanding of robot components and basic logic)

## Level Structure

### Level 1: Getting Started with Blocks (Weeks 1-2)
Introduction to block programming environment and simple movement sequences.

**Modules:**
- Block Basics (interface, running programs, connecting to robot)
- Simple Sequences (forward, turn, sound, movement combinations)

**Assessment**: Successfully run 5 different movement sequences

**Project**: Create a robot dance with 10+ movements

### Level 2: Loops and Repetition (Weeks 2-3)
Making robots repeat actions efficiently.

**Modules:**
- Loop Fundamentals (repeat loops, forever loops, loop counters)
- Practical Loop Applications (line following patterns, repeated behaviors)

**Assessment**: Design a program that efficiently moves robot in square

**Project**: Create a robot that traces a 5-shape pattern using loops

### Level 3: Conditionals and Decisions (Weeks 4-5)
Programming robots to make choices based on sensor input.

**Modules:**
- If-Then Statements (basic conditional logic, if-else, nested conditions)
- Sensor-Based Decisions (touch sensors, color sensors, distance sensors)

**Assessment**: Program obstacle avoidance using distance sensor

**Project**: Create an obstacle avoider robot

### Level 4: Events and Advanced Programming (Weeks 5-6)
Event-driven programming and complex behaviors.

**Modules:**
- Event-Driven Programming (button events, sensor events, multiple handlers)
- Complex Behaviors (combining loops, conditionals, events)

**Assessment**: Event-driven program with multiple behaviors

**Capstone**: Interactive robot controlled by multiple sensors and buttons

## Notes for Content Developers

### Pedagogical Approach
1. **Concrete Before Abstract**: Show blocks on screen, discuss meaning
2. **Immediate Feedback**: Run code immediately to see results
3. **Progressive Complexity**: Simple movements → sequences → loops → conditionals → events
4. **Hands-On**: Always have physical robot or simulator
5. **Discovery**: Let students experiment before formal instruction

### Common Challenges
- Students rushing through without understanding
- Difficulty translating thinking into blocks
- Debugging skills development
- Performance anxiety around coding

### Misconceptions to Address
1. Blocks execute simultaneously (they don't; they're sequential)
2. Variables automatically update (they need reassignment)
3. Events and blocks are the same thing
4. Loops always make robots go in circles
5. If-Then means "sometimes" not "always when true"

### Debugging Strategies
- Trace program execution step-by-step
- Add debug outputs (sounds, lights)
- Test individual blocks before combining
- Use print/speak blocks for troubleshooting
- Predict behavior before running
